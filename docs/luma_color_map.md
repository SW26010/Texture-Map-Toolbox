# 亮度色彩映射 (Luma Color Map)

## 概述

从一张彩色图像出发，通过亮度值建立颜色映射关系，生成 256 级亮度-颜色查找表 (LUT)。

## 原理

### 输入

一张带 Alpha 通道的彩色 PNG 图像。

### 处理流程

1. **图像加载与预处理**
   - 提取 RGB 和 Alpha 通道，将图像转为浮点格式 (0-1)
   - 使用有效掩码 (`valid_mask`) 标识非透明像素

2. **灰度转换**
   - 使用 ITU-R BT.709 标准计算亮度：
     $$Y = 0.2126R + 0.7152G + 0.0722B$$

3. **建立亮度-颜色查找表 (LUT)**
   - 遍历灰度值 0–255
   - 对每个灰度值 $g$，在灰度图中找出所有等于 $g$ 的像素
   - 计算这些像素在原图中的 R、G、B 平均值
   - 使用 `np.bincount()` 加权求和高效完成计算
   - 生成 256×3 的查找表

4. **颜色空间转换**
   - 将 RGB LUT 转换为 HSL 颜色空间
   - 对 NaN 值（灰度值在图像中未出现的情况）进行处理

5. **重建与质量评估**
   - 根据灰度值从 LUT 查表，重新着色为彩色图像
   - 计算 PSNR（峰值信噪比）评估像素级差异
   - 计算 CIEDE2000 色差（Delta E 2000）评估人眼感知色差
   - 统计 Delta E 的均值、中位数、标准差、最大值、95 百分位数

### 输出

- 256×3 的亮度-颜色查找表
- 重建后的 RGB 图像（8-bit 量化）
- 质量指标（PSNR、Delta E 统计值）

## 可视化

脚本会生成两组图表：

**对比图（4 宫格）**
| 面板 | 内容 |
|------|------|
| 1 | 灰度图 |
| 2 | 原始图像 |
| 3 | 重建图像 + PSNR |
| 4 | CIEDE2000 误差热力图 |

**分析图（5 子图）**
| 子图 | 内容 |
|------|------|
| 1 | 亮度像素分布直方图（对数坐标）|
| 2 | 亮度 → Lightness (L*) 曲线 |
| 3 | LUT 颜色条可视化 |
| 4 | 色相 (Hue) 通道 + 色轮背景 |
| 5 | 饱和度 (Saturation) 曲线 |
